# OSA 打鼾干预实验 · 使用说明

> 这一份给所有要跑实验的人（被试 / 协作者 / 研究员）。  
> 全程只需要双击三个 `.command` 文件。

---

## 一、需要什么

- **macOS 13+** 的 Mac（推荐 Apple Silicon / M 系列）
- **任意蓝牙耳机**（AirPods / 荣耀 / 任意品牌；放干预音用）
- **HSR-1A2.0 胸带**（戴胸前；测姿态 / 呼吸 / 心电 / 转发血氧）
- **PC-68B 指夹血氧仪**（开机夹手指即可，**不需要单独连蓝牙**，数据自动经胸带 BLE 进来）
- **充电线**：整晚保持插电

---

## 二、第一次安装（10 分钟，只做一次）

```bash
git clone https://github.com/xuxu8421/osa.git ~/osa
cd ~/osa/dist
```

打开 Finder 找到 `~/osa/dist/`，**双击 `setup.command`**。

如果弹出 "无法打开，因为不是 App Store 下载的…"：右键点该文件 → **打开** → 再次确认。（macOS 之后会记住。）

终端窗口会自动跑：
- 检测 / 创建 venv 虚拟环境
- 下载依赖（**TensorFlow 包比较大，首次 5–10 分钟**）
- 下载 YAMNet 鼾声检测模型 (~15 MB → `~/.cache/osa_yamnet/`)
- 列出你电脑上的音频设备

看到 `✅ 安装完成!` 就关掉窗口。

> 不会 git 也行：去 GitHub 仓库右上角点 `Code → Download ZIP`，解压到桌面后进 `dist/` 双击 `setup.command`，效果一样。

---

## 三、每晚开始（2 分钟）

1. **耳机** 用蓝牙连上 Mac。
2. **胸带**戴好（电极湿一下贴胸前），**血氧仪**开机夹手指。
3. 双击 **`start_night.command`**
   - 浏览器自动打开 `http://localhost:8000`
   - 这个终端窗口**整夜保持打开**（可以最小化）
4. 在浏览器里：
   - **设备连接 tab**：点「扫描」→ 列表里选 HSR/1A2/SRG → 点「连接」
   - 下方「音频通道」确认输入是 MacBook 麦克风、输出是你的耳机（耳机不在列表里就左上角蓝牙菜单点一下耳机让 macOS 唤醒，再点网页「重新枚举设备」）
   - 回到 **实时实验 tab**，顶部填 **被试 ID**、**备注**，点「**开始会话**」
5. 戴好耳机，关灯上床。

---

## 四、第二天早上（3 分钟）

1. 起床后回到那个还开着的终端窗口，按 **Ctrl-C**（或直接 ⌘W 关）。
2. 双击 **`end_night.command`**。  
   它自动：结束会话 → 跑分析脚本 → 把 `sessions/<日期>/` 整个打包成 zip → 弹出 `export/` 文件夹。
3. **把 `export/osa-data-xxx.zip` 发给研究员**（微信 / 飞书 / AirDrop 任选）。

## 五、网页上的几个关键 UI

- **顶部 6 个 badge**：`会话进行中 / 闭环已启用 / 姿态:仰卧 / 检测到打鼾 / 不在冷却中 / 在活跃时段内` —— 全绿即正在 armed，持续 8 秒会播一次干预音。
- **顶部状态背板颜色**：灰=待机；绿=条件满足中；红闪=正在播放；蓝=观察期；深灰=冷却。
- **鼾声判决时间线**：蓝线 = 打鼾概率，绿色阴影 = 判为打鼾的时段，橙色虚线 = 阈值。
- **回放 / 审核 tab**（事后看）：每次干预的 ±60 秒胸呼吸 / SpO2 / 鼾概率 / 麦克录音 / 干预音都能听能看。

## 六、可选：每晚开始前自检

双击 **`preflight_check.command`**：列音频设备 + 验证 YAMNet + 放 3 声"叮"。  
任意一项失败就在终端里看错误信息，再处理。

## 七、出问题怎么办

| 症状 | 怎么处理 |
|---|---|
| 浏览器打不开 `localhost:8000` | 看 start_night 的终端有没有报错；关掉再双击一次 |
| 耳机没出现在输出下拉里 | 左上角蓝牙菜单点一下耳机 → 网页点「重新枚举设备」 |
| 胸带连不上 / 断线 | 「设备连接」tab 点断开再扫描重连 |
| 顶部 SpO2 黄色"失效 (查 PC-68B)" | PC-68B 没开 / 没夹紧 / 没电；它是 SpO2 唯一来源 |
| `检测到打鼾` 一直不亮 | 对着 MacBook 麦克风假装打鼾两声看概率；不行就在「鼾声检测阈值」面板把阈值降到 0.2 |
| 自动触发一直没发生 | 顶部 6 个 badge 哪个没绿？通常是没点「开始会话」或当前不在「活跃时段」内 |
| 装包失败 | 在终端里：`cd ~/osa && bash dist/setup.command`，看具体报错 |
| YAMNet 加载报错 | 删 `~/.cache/osa_yamnet/`，再跑 `bash dist/setup.command` |

## 八、想改干预参数

进 `http://localhost:8000` → 实时实验 tab → 展开「**控制器阈值**」：

| 旋钮 | 默认 | 含义 |
|---|:-:|---|
| 仰卧持续 → 触发 | 8 s | 仰卧 + 鼾声满足多久才触发首次干预 |
| 额外确认鼾声次数 | 1 | 首次触发前还需要听到几次新的鼾声（防误触发） |
| 连发重试武装 | 2 s | 干预无响应后再次蓄力的短时长 |
| 冷却 · 成功后 | 180 s | 翻身成功后的静默期 |
| 冷却 · 无反应后 | 5 s | 没反应时的快速重试间隔 |
| 播放响度 | -15 dB | 太大被试醒、太小没反应 |
| 活跃时段 | 全程 | 例如设 `01:00 → 04:30` 只在这段时间干预 |

「**鼾声检测阈值**」面板：

| 旋钮 | 默认 | 何时改 |
|---|:-:|---|
| Snoring 概率阈值 | 0.30 | 安静环境 0.20-0.25；嘈杂环境 0.4-0.5 |

## 九、给研究员：收到数据后怎么看

```bash
# 解压到 sessions/ 下
cd ~/osa/sessions && unzip ~/Downloads/osa-data-xxx.zip

# (可选) 重跑分析
cd ~/osa && python3 scripts/analyze_night.py sessions/<日期>

# 起 Web，去「回放 / 审核」tab 看每个触发的多通道波形 + 音频
python3 run_designer.py --web
```

`sessions/<id>/` 目录里有：

```
meta.json                会话元信息 (subject, config)
summary.json             简短统计
events.jsonl             所有事件
interventions.jsonl      每次干预记录
chestband.csv            每秒 vitals
chestband_####.npz       胸带原始波形分块
events/
├── <ts>_A_<strategy>_<dir>.npz       ±30s 多通道快照
├── <ts>_A_<strategy>_<dir>_mic.wav   ±10s 麦克录音
└── <ts>_A_<strategy>_<dir>_played.wav 当次播放的干预音
report/
├── strategy_report.csv  每次一行: 策略 / 成功 / 潜伏 / SpO2 改善
└── strategy_report.md   人看的汇总表
```
