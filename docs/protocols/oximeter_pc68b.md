# PC-68B 血氧仪 · BLE 笔记 (可选设备)

> **重要**：胸带 1A2.0 自己就提供实时 SpO2 和脉搏波，PC-68B 是**可选冗余**。
> 不连也能跑 Block A。本文档仅在有需要交叉验证 SpO2 时参考。

## 厂商协议

PC-68B 使用私有 GATT 协议，一些 characteristic 的具体含义未公开。我们在
`devices/oximeter.py` 里做了**半逆向**的解析：通过抓帧 + 经验对齐字段。
每一帧通常包含：

- SpO2 % (0-100)
- 脉率 bpm
- PI (Perfusion Index) 灌注指数 %
- 脉搏波 pleth 字节
- 状态标志：`finger_out`（手指未插入）、`probe_error`（探头异常）

## 连接流程

- `OximeterBLE.scan(oximeter_only=True)`：按设备名关键字筛选
- `connect(device)` 后订阅 NOTIFY；某些型号需要先**写入唤醒帧**才开始推流
- UI "调试" tab 有手动 HEX 输入框，用于抓包时试验命令序列

## 兼容性

- PC-68B 旧固件行为相对稳定
- 同系列（PC-60FW, PC-80B 等）字段布局接近但不完全一致，目前代码只针对
  PC-68B 做过验证

## 设计决策：为什么还保留这个设备

1. 胸带 SpO2 测量部位通常在手腕或胸前（夹式），PC-68B 在手指端，两个部位
   的数据互为**生理冗余**
2. 实验中偶有胸带脱位 → SpO2 失准，此时可用 PC-68B 继续
3. 将来做数据质量审核时，两路一致性是有价值的指标
