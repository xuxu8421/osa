# 模式 S · 仅姿态干预（简化版）

**目标**：在不戴血氧仪、不开麦克风的前提下，验证「仰卧持续一段时间 → 播放
带方向干预音」是否能让被试主动翻身离开仰卧。  
适合开发自验、初步可行性测试，或在没有 PC-68B 血氧仪的场景下做体位干预的纯
独立实验。

## 单夜对比多种声音 · 轮播模式

S 模式的默认配置就是为「单夜对比多策略」设计的：

```
默认: pool = [P1, P2, P3]   rotation_min = 60
```

控制器会按会话开始时间分段，每 60 分钟用一种策略，到点自动切到下一种，
最后一种放完回到第一种循环：

```
t = 0  – 60 min   P1
t = 60 – 120 min  P2
t = 120 – 180 min P3
t = 180 – 240 min P1   (回到第一种，开新一轮)
...
```

被试只跑一晚就能拿到 3 种策略各 N 次的"成功翻身率 / 潜伏 / 触发频率"，
分析时按 `interventions.jsonl` 里的 `strategy` 字段直接分组对比，不用安排
多晚。

可以在 UI 的策略卡片里：
- 勾选要进入轮播的策略集合（默认全勾 P1/P2/P3）
- 用下拉框改轮播间隔（15 / 30 / 45 / 60 / 90 / 120 分钟）
- 想退回"完全随机"模式就把间隔设为 `0`
- 想整夜固定一种就只勾 1 个（间隔自动失效，因为单元素没的轮）

> 想中途换轮播配置请先「结束会话」再改 — 防止同一晚的数据被两套配置
> 污染。

## 与 Block A 模式的对照

| 维度 | A · 仰卧+鼾声 (Block A) | **S · 仅姿态 (本模式)** |
|---|---|---|
| 必要硬件 | 胸带 + 蓝牙耳机 + PC-68B 血氧仪 + Mac 麦克风 | **仅胸带 + 蓝牙耳机** |
| 触发条件 | 仰卧 ∧ 最近 15 s 内有打鼾 | **仅仰卧持续 X 秒** |
| 触发持续 (默认) | 8 s | **15 s**（更保守，无鼾声做门控） |
| 观察窗 (默认) | 10 s | **15 s** |
| 无反应冷却 (默认) | 5 s | **8 s** |
| 后台 YAMNet | 持续推理 | **暂停**（关麦克风采集） |
| 会话标记 | `meta.mode = 'A'` · `protocol = 'block_a_pilot'` | `meta.mode = 'S'` · `protocol = 'posture_only_pilot'` |
| 干预策略池 | P1/P2/P3 随机 | 同（不变） |
| 方向策略 | `opposite`（仰卧时随机选 L/R） | 同（不变） |

## 状态机

简化后只剩 4 个阶段（去掉了"等鼾声满足"的逻辑）：

```
等待中 ─[仰卧持续 15 s]─▶ 播放干预音 ─▶ 观察 15 s ─┬─[翻身]─▶ 成功后冷静 180 s ─▶ 等待中
                                                  └─[超时]─▶ 无反应 8 s 短冷却 ─▶ 等待中
```

> 与 Block A 完全相同：随机抽 P1/P2/P3 之一，方向取仰卧反侧（仰卧时随机
> 左/右），用空间化 ITD/ILD 让左右耳有差异。

## 怎么进入这个模式

### Web UI

启动 `python3 run_designer.py --web` 之后，浏览器顶栏会有两个标签页：

```
[ A · 仰卧+鼾声 ]  [ S · 仅姿态 ]
```

点 `S · 仅姿态` 即切换。系统会：

1. 立刻把控制器阈值改成 S 模式默认值（hold 15s / observe 15s / 短冷却 8s）。
2. 把 `require_snoring` 强制设为 false、`confirm_snore_bouts` 设为 0。
3. 关掉麦克风输入流和 YAMNet 推理（节省 CPU、不占用麦克风）。
4. UI 自动隐藏「检测到打鼾」徽章、鼾声判决时间线、YAMNet 阈值面板。
5. SpO2 / 脉率两张数据卡灰显并标注「S 模式 · 未启用」。

> 切回 A 模式同理：自动恢复 A 默认值并重新启动 YAMNet。

### REST

```
POST /api/experiment/mode    {"mode": "S"}    # 切到 S
POST /api/experiment/mode    {"mode": "A"}    # 切回 A
```

成功响应携带切换后的完整控制器配置；失败会带 `err` 字段（最常见的失败：
当前会话进行中，需要先 `POST /api/session/stop`）。

## 数据落盘

落盘格式与 Block A 一致（`sessions/<id>/`），区别只在标记字段：

```
meta.json           "mode": "S",  "protocol": "posture_only_pilot",
                    "config": { "require_snoring": false, "trigger_hold_s": 15.0, ... }

interventions.jsonl 每行带 "block": "S"
events/<ts>_S_<strategy>_<dir>.npz                ±30 s 胸呼吸 + ±60 s SpO2 (空)
events/<ts>_S_<strategy>_<dir>_played.wav         当次播放的干预音
events/<ts>_S_<strategy>_<dir>_mic.wav            ❌ 不会生成（麦克风未启动）
```

`scripts/analyze_night.py` 同样能跑，输出的 `strategy_report.csv` 里"前/后
30 s 鼾声占比"两列会全部为空（没采集鼾声），其余指标（成功率、潜伏、SpO2
变化）正常生成（SpO2 那列在 S 模式下也会空）。

## 默认参数依据

S 模式没有"打鼾"作为额外门控，单靠"仰卧持续 X 秒"就要决定开火。8 s 太
短：被试在体位间过渡、短暂仰卧再翻走的场景很常见，会出现假触发。15 s 是
经验上仰卧"已经稳定坐定"的最短判定值。其他时长按 1.5–2× A 模式调整。

如果觉得太保守，可以在 UI 的「控制器阈值」面板拉到 5–10 s。
