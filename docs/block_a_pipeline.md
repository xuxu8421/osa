# Block A · 体位干预 · 状态机

**目标**：被试仰卧且持续打鼾时，播放一个带方向的空间化声音（从仰卧的反侧
播），诱导其侧卧；不求唤醒。

## 状态表

| 阶段 | 进入这个阶段的条件 | 这个阶段在做什么 | 停留时长 | 下一个阶段 |
|------|------------------|----------------|---------|-----------|
| **等待中** | 刚开始；或上一次干预结束后冷静期到了 | 持续监听姿态和鼾声，不做任何干预 | 没有上限，直到下一条触发条件满足 | 受试者仰卧 且 最近 15 秒内检测到过打鼾 → **条件满足中** |
| **条件满足中** | 受试者仰卧 且 最近 15 秒内有打鼾 | 开始计时，等条件持续足够久 | 条件持续 **8 秒** 后进入下一步；期间只要离开仰卧，或鼾声超过 15 秒没再出现，就回到"等待中" | 继续满足 8 秒 → **播放干预音** |
| **播放干预音** | 上一步的 8 秒条件持续达成 | 从 P1 / P2 / P3 里**随机抽一个**，按方向策略选左/右，合成并播放给 AirPods | 每次播放 2 – 4 秒 | 播放结束 → **观察反应** |
| **观察反应** | 干预音刚播完 | 盯住胸带看被试是否翻身，不做其他事 | 最多 **10 秒** | 在 10 秒内姿态变化了 → **成功后冷静**；10 秒没变化 → **无反应后稍等** |
| **成功后冷静** | 观察期内发现被试翻身离开仰卧 | 强制不打扰，让他安稳睡 | **180 秒** | 冷静结束 → **等待中** |
| **无反应后稍等** | 观察期 10 秒都过去了被试没动 | 短暂停顿 | **5 秒** | 停顿结束 → **等待中**（如果条件仍满足会立刻进入下一轮） |

## 两个细节设计

### 1. "最近有打鼾"而不是"现在正在打鼾"

打鼾是阵发性的，一次呼吸周期 3–6 s，相邻两次鼾声之间有明显间隙。如果判据
用**当前瞬时 `is_snoring()`**，8 秒累计几乎永远达不到。因此控制器维护
`_last_snore_at`：**只要最近 15 秒内出现过一次"正在打鼾"，就视为鼾声条件
满足**。15 秒这个窗口可以在 UI 里调。

### 2. 成功与无反应走不同长度的冷却

- **成功翻身**：被试明确动了，接下来几分钟他在自己安稳地换姿势，我们没必要
  再干预。**冷却 180 秒**。
- **无反应**：干预音可能不够、或策略没选对、或被试已经醒了但没翻身。既然
  气道阻塞仍在，应当**快速重试**。**冷却仅 5 秒**，条件如果还满足会立刻重新
  累计触发。

## 时间轴示例

### 示例 A · 一次成功的干预

```
t=0.0 s    受试者已仰卧 5 s，此刻开始出现打鼾                → 等待中
t=0.1 s    snoring_prob 首次 > 0.3                         → 条件满足中 (计时 0/8)
t=8.2 s    持续 8 s                                        → 播放干预音
t=8.3 s    随机选中 P2；方向=right（仰卧随机选）              →
t=8.3 s    开始播放 P2 right (约 3s)                       → 播放干预音
t=11.3 s   播放完毕                                         → 观察反应 (10 s 窗口)
t=14.6 s   胸带姿态 supine → left (被试翻身!)                → 成功后冷静 (180 s)
t=194.6 s  冷却结束                                          → 等待中
```

### 示例 B · 两次连续的无反应干预

```
t=0.0 s    仰卧 + 打鼾                                → 条件满足中
t=8.0 s    持续 8 s                                   → 播放 P1 left
t=8–11 s   播放中                                     → 播放干预音
t=11.0 s   播完                                      → 观察反应
t=21.0 s   10 s 观察完未翻身                          → 无反应后稍等 (5 s)
t=26.0 s   冷却结束，条件仍满足 → 立即                 → 条件满足中
t=34.0 s   再次累计到 8 s                              → 播放 P3 right
...
```

即 **一轮"无反应循环"节拍约 26 秒**（播放 ~3 s + 观察 10 s + 短冷却 5 s +
重新 armed 8 s）。如果希望更激进，可以缩短 `trigger_hold_s` 或
`response_window_s`。

### 示例 C · 条件中途失效

```
t=0 s     仰卧 + 鼾声                   → 条件满足中
t=3 s     受试者翻身到 left              → 等待中 (计时重置，不触发)
```

鼾声中间有 10 s 空档但 15 s `snoring_recent_s` 窗口未过期时，计时器继续累加
（这就是"最近有过鼾声"窗口的意义）。

## 可调参数

| 参数 | 默认 | 作用 |
|------|------|------|
| `trigger_hold_s` | 8 s | 条件持续多久才播 |
| `snoring_recent_s` | 15 s | "最近有打鼾"算多久 |
| `response_window_s` | 10 s | 观察期长度 |
| `cooldown_s` | 180 s | 成功翻身后的长冷却 |
| `cooldown_no_response_s` | 5 s | 无反应后的短冷却 |
| `level_db` | -15 dB | 播放响度 |
| `direction_policy` | `opposite` | `opposite` / `left` / `right` / `random` |
| `strategy_pool` | `(P1,P2,P3)` | 随机池；改成 `(P1,P2,P3,L1,L2)` 就都参与 |
| `active_window_start / end` | `''` | HH:MM；非空时仅在时段内工作，支持跨午夜 |

所有这些都能在 Web UI 的「控制器阈值」展开面板里拖滑块现调现生效，不用重启。

## 代码定位

- 状态机：`pipeline/controller.py::ClosedLoopController`
- 活跃时段判断：`ClosedLoopController._within_active_window`
- 鼾声"最近有过"逻辑：`ClosedLoopController._snoring_ok`
- 成功/无反应分支：`_on_posture_change`（成功） + `_play_then_observe` 的
  超时分支（无反应）
- 策略选择：`_fire` 里 `random.choice(strategy_pool)` + `_pick_direction`
